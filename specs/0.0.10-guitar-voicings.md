# MCC 0.0.10 — Guitar Voicings Design Specification

Design document for the next MusicCraftCore release. Diagnosis-plan-execute per Tier 1 discipline.

Status: Ready for implementation.

---

## Purpose

0.0.10 ships the **Instruments/Guitar/** subsystem: voicing library, capo position suggestions, and voicing scoring with composable criteria. This unblocks **Sanctuary slice 9.3** (vocal harmony suggestions), where a singer captures a sung melody, MCC infers the key and suggests diatonic chord candidates, and the UI renders tappable voicings with optional capo positions for self-accompaniment.

The originating context: Cantus pivoted from chord-detection-on-vocals (wrong problem class for monophonic input) to vocal harmony suggestions (rule-based diatonic candidates). MCC ships the voicing data and math layer; Sanctuary owns rendering.

## Capability-Context Fit (reproduced from prompt)

**Component:** `VoicingLibrary` takes a `Chord` value + a `GuitarTuning` and returns ranked playable voicings. Each voicing assumes a 6-string fretted instrument tuned to a defined open-string semitone pattern. `CapoCalculator` takes a target `MusicalKey` and returns capo positions that map voicing-rich source keys to it.

**Destination:** Sanctuary slice 9.3 (vocal harmony suggestions). Input is a sung monophonic melody; the user-facing outcome is "Cantus suggests these chords, here's how to play each one with optional capo positions." The destination renders tappable shapes; it does not detect chords from audio.

**Match:** Yes. Chord-name → playable shape is exactly what slice 9.3 needs. Voicing data is proven in legacy Cantus production. Capo math is additive guitar-domain. No problem-class drift.

**Sub-audit on scoring:** Voicing scoring is genuinely subjective. The API exposes composable criteria with overridable weights, not a single opinionated score. Consumers don't read "best voicing" into "highest score."

## Scope

**In scope:**
- New `Sources/MusicCraftCore/Instruments/Guitar/` subsystem with 6 public types
- `GuitarTuning` — 6 standard tunings (EADGBE, Drop D, Open D, Open G, DADGAD, CGDGBD)
- `VoicingPosition` — fretboard shape (frets, fingers, barres, baseFret, requiresCapo)
- `GuitarVoicing` — position + chord metadata with computed displayName
- `VoicingLibrary` — public enum; `voicings(for:tuning:limit:) -> [GuitarVoicing]`
- `CapoCalculator` — public enum; `suggestions(targetKey:tuning:maxCapoFret:) -> [CapoSuggestion]`
- `CapoSuggestion` — sourceKey, capoFret, targetKey, score
- `VoicingScore` — fingeringDifficulty, openness, positionScore, spanScore, totalScore
- `VoicingScoringCriteria` — composable weights (default: 0.4 difficulty, 0.3 openness, 0.2 position, 0.1 span)
- Curated `guitar_voicings.json` resource (72 chord-name keys × 2–3 voicings)
- Comprehensive test suite covering all types and algorithms
- CHANGELOG entry, CODEX update, Version.swift bump to 0.0.10

**Out of scope (deferred):**
- Diminished and augmented voicings (gaps in diatonic vii° major / ii° minor — deferred to 0.0.11)
- Per-tuning voicing libraries (alternate tunings ship as data but VoicingLibrary returns empty in v1)
- Chord substitution suggestions
- User-defined voicings
- Left-handed mirroring

## API Surface

### GuitarTuning

```swift
public enum GuitarTuning: String, CaseIterable, Equatable, Hashable, Sendable, Codable {
    /// Standard tuning: EADGBE (low E to high E)
    case standard = "Standard"
    case dropD = "Drop D"
    case openD = "Open D"
    case openG = "Open G"
    case dadgad = "DADGAD"
    case cgdgbd = "CGDGBD"
    
    /// Identifer (lowercased, no spaces)
    public var id: String
    
    /// Human-readable name (e.g., "Standard", "Drop D")
    public var displayName: String
    
    /// Short name for compact display (e.g., "Std", "DD")
    public var shortName: String
    
    /// Semitone intervals from C0 (low to high string), e.g. Standard = [40, 45, 50, 55, 59, 64] for E A D G B E
    public var semitones: [Int]
    
    /// Reference frequencies in Hz for each open string (A4 = 440 Hz, 12-tone equal temperament)
    public var referenceFrequencies: [Double]
    
    /// Human-readable description of the tuning
    public var description: String { "..." }
}
```

All cases Equatable, Hashable, Sendable, Codable. Static catalog with `.standard` as the default for most lookups.

### GuitarVoicing

```swift
public struct GuitarVoicing: Equatable, Hashable, Sendable, Identifiable {
    /// Unique identifier
    public let id: UUID
    
    /// Chord this voicing represents
    public let chord: Chord
    
    /// Tuning this voicing is designed for
    public let tuning: GuitarTuning
    
    /// Fretboard shape
    public let position: VoicingPosition
    
    /// Computed display name (e.g., "C — open position" or "F — barre at 1")
    public var displayName: String {
        let posDesc = position.baseFret == 1 ? "open" : "fret \(position.baseFret)"
        return "\(chord.displayName) — \(posDesc)"
    }
    
    public init(id: UUID = UUID(), chord: Chord, tuning: GuitarTuning, position: VoicingPosition)
}
```

### VoicingScore

```swift
public struct VoicingScore: Equatable, Hashable, Sendable {
    /// Fingering difficulty (0–1, lower = easier). 1 - (uniqueFingers/4 + barre_penalty + span_penalty)
    public let fingeringDifficulty: Double
    
    /// Openness (0–1, higher = more open strings)
    public let openness: Double
    
    /// Position score (0–1, higher = lower on fretboard)
    public let positionScore: Double
    
    /// Span score (0–1, higher = tighter fret span)
    public let spanScore: Double
    
    /// Weighted total score (0–1)
    public let totalScore: Double
    
    public init(
        fingeringDifficulty: Double,
        openness: Double,
        positionScore: Double,
        spanScore: Double,
        totalScore: Double
    )
}

public struct VoicingScoringCriteria: Equatable, Hashable, Sendable {
    /// Weight for fingering difficulty (0–1)
    public let weightDifficulty: Double
    
    /// Weight for openness (0–1)
    public let weightOpenness: Double
    
    /// Weight for position (0–1)
    public let weightPosition: Double
    
    /// Weight for span (0–1)
    public let weightSpan: Double
    
    /// Default criteria: 0.4 difficulty, 0.3 openness, 0.2 position, 0.1 span
    public static let `default`: VoicingScoringCriteria
    
    public init(
        weightDifficulty: Double,
        weightOpenness: Double,
        weightOpenness: Double,
        weightPosition: Double,
        weightSpan: Double
    ) throws  // Validates weights sum to 1.0 ± 0.001
}

/// Score a voicing using optional custom criteria (default: VoicingScoringCriteria.default)
public func score(_ voicing: GuitarVoicing, criteria: VoicingScoringCriteria = .default) -> VoicingScore
```

## Algorithm — Capo Math

```
For target key K with maxCapoFret M:
  1. Determine source-key pool based on mode:
     - major target → [C major, G major, D major, A major, E major]
     - minor target → [A minor, E minor, D minor]
  2. For each source key S:
     capoFret = (K.root.rawValue - S.root.rawValue + 12) % 12
     if capoFret ≤ M:
       score = sum of voicing.count for all diatonic chords in S
       candidate (S, capoFret, K, score)
  3. Sort candidates by score descending; ties broken by lower capoFret ascending
  4. Return top 3 (or fewer if fewer exist)
```

**Mode note:** Major and minor are mode-preserved. Relative major↔minor mapping is deferred to 0.0.11. This simplifies the algorithm and matches the singer's likely key context (if inferring key from their melody, relative mode mapping adds complexity that can land later).

## Algorithm — Voicing Scoring

```
Given a voicing (position with frets and fingers):

fingeringDifficulty:
  base = (uniqueFingersUsed / 4) + 0.2 * barres.count + 0.2 * (fretSpan > 4 ? 1 : 0)
  clamp to [0, 1]
  Return (1 - base) as the difficulty score (higher = easier)

openness:
  count(frets[i] == 0) / 6
  Range [0, 1]

positionScore:
  1 - (baseFret / 12), clamped to [0, 1]
  Range [0, 1]

spanScore:
  fretSpan = max(fret for fret in frets if fret >= 0) - min(fret for fret in frets if fret >= 0)
  1 - min(fretSpan / 4, 1)
  Range [0, 1]

totalScore:
  w_fd*(1-fingeringDifficulty) + w_o*openness + w_p*positionScore + w_s*spanScore
  (Weights from VoicingScoringCriteria; must sum to 1.0)
```

**Anchor note:** Default weights (0.4 difficulty, 0.3 openness, 0.2 position, 0.1 span) are tuned for an open-position-preferring acoustic singer-songwriter use case (Chris's profile). Consumers with different goals (fingerstyle, jazz, rock) override via custom VoicingScoringCriteria.

## Curation Criteria for Bundled JSON

Source: legacy Cantus `guitar_voicings.json`

**Coverage:** 72 chord-name keys
  - Roots: C, C#, D, D#/Eb, E, F, F#, G, G#/Ab, A, A#/Bb, B (12 total)
  - Qualities: major (""), minor ("m"), maj7, 7, sus2, sus4 (6 total)

**Selection process per (root, quality) combination:**
  1. List all candidate voicings from legacy
  2. Sort by `(baseFret ascending, openStringCount descending)`
  3. **Rank 1:** Top of sort (canonical open or lowest-position voicing)
  4. **Rank 2:** Lowest-baseFret voicing with `requiresCapo=true` (movable barre form). If none, pick next-lowest-baseFret not already chosen.
  5. **Rank 3:** Only for "easy" keys (C, G, D, A, E, Am, Em, Dm, Bb, F + their maj7/7 variants = ~20 chord names). Pick an alternate position with distinctly different baseFret or openness. Skip if no clear alternate.

**JSON shape:**
```json
{
  "_meta": {
    "version": "0.0.10",
    "source": "legacy Cantus guitar_voicings.json (curated subset)",
    "curated_on": "2026-05-08",
    "criteria": "12 roots × 6 qualities × 2–3 voicings each. Rank 1: lowest baseFret + most opens. Rank 2: lowest-baseFret barre. Rank 3: alternate for easy keys.",
    "chord_count": 72,
    "voicing_count": 168
  },
  "voicings": {
    "C": [
      { "frets": [...], "fingers": [...], "baseFret": 1, "requiresCapo": false },
      ...
    ],
    "Cm": [...],
    ...
  }
}
```

**Field mapping:**
  - Legacy `capo: bool` field → new `requiresCapo: bool` field in JSON
  - Barres omitted (not null) when not present, matching legacy convention

**Chord name format:** ASCII notation. Major roots with no quality suffix ("C"), sharp with "#" ("C#"), flat with "b" (used only in aliases; we prefer sharp internally), minor with "m", maj7/7/sus2/sus4 as shown in API surface.

## Test Plan

Create `Tests/MusicCraftCoreTests/Instruments/Guitar/` with one test file per source:

1. **GuitarTuningTests**
   - Verify all 6 catalog cases present
   - Verify semitone-to-frequency round-trip for standard tuning (A4 = 440 Hz baseline)
   - Verify Codable round-trip

2. **VoicingPositionTests**
   - Verify Codable round-trip
   - Verify frets array length validation (must be 6)
   - Verify legacy `capo` JSON field decodes to `requiresCapo` (defensive)

3. **GuitarVoicingTests**
   - Verify displayName generation (representative chords: Am, F#m7, Bbsus4)
   - Verify Identifiable id stability

4. **VoicingLibraryTests**
   - Bundled JSON loads without error
   - Returns non-empty voicings for every curated chord (72 total)
   - Returns empty for unknown chord
   - Returns empty for non-standard tuning (expected in v1)
   - Respects limit parameter
   - Standard tuning lookups are deterministic

5. **CapoCalculatorTests**
   - Known mappings: G major + maxCapo 7 → C major capo 5; Eb major + maxCapo 7 → C major capo 3, D major capo 1
   - Target B minor + maxCapo 7 → A minor capo 2
   - Mode separation: B minor returns no B major sources (relative mode mapping deferred)
   - Scoring orders by diatonic richness (more voicings = higher score)
   - Respects maxCapoFret

6. **VoicingScoreTests**
   - Open Am scores higher openness than barre Am
   - F major barre at fret 1 scores lower position than C major at fret 0
   - Default criteria weights sum to 1.0
   - Custom criteria with overrides produce different rankings than default

All tests passing before push. Pre-push hook validates against `.git-hooks/known-failing-tests.txt`. No new failures allowed.

Every ported type maps to legacy origin:

- `VoicingPosition` ← `Cantus/Assistant/ReferenceDataModels.swift` lines 7–13 (struct VoicingPosition)

Dropped from v1:
- Easter egg tunings (not in standard catalog)
- UserDefaults persistence (app-side if needed)

## Out of Scope

Explicitly deferred:

- Diminished and augmented voicings (vii° major / ii° minor diatonic gaps in bundled data)
- Cross-mode capo mapping (relative major↔minor for transposition)
- Alternate-tuning voicing libraries (data shipping but VoicingLibrary returns empty in v1)
- Chord substitution suggestions
- User-defined voicings
- Left-handed mirroring
